# 📚 迭代信息 - 从漫画阅读器到小说阅读器

## 项目概述

本项目从基于 Tauri 2 + React + Rust 的漫画阅读器改造为小说阅读器，参考 Koodo-Reader 开源项目的技术理念和功能设计。

### 改造目标
- 保留现有 60-70% 的架构和代码
- 支持主流文本格式（PDF、TXT、MD）
- 提供舒适的文本阅读体验
- 支持书签、高亮、目录导航等核心功能

### 技术栈变更
| 层级 | 改造前 | 改造后 |
|------|--------|--------|
| 桌面框架 | Tauri 2 | Tauri 2（保留） |
| 前端框架 | React 18 + TypeScript | React 18 + TypeScript（保留） |
| 状态管理 | Zustand | Zustand（保留） |
| 数据库 | SQLite | SQLite（保留） |
| 核心解析 | zip/unrar/lopdf | pdf-extract/chardet/marked + MD 解析 |
| 渲染引擎 | pdfjs-dist + 图片渲染 | 新增文本渲染引擎 |

## 🎯 关键技术决策

### 1. PDF 文本提取方案
- **选项 A**：继续使用 lopdf，改为文本提取模式
- **选项 B**：使用 pdf-extract crate
- **建议**：先用 lopdf 快速实现，后续按需切换到 pdf-extract

### 2. TXT 编码检测方案
- **方案**：chardet 自动检测 + encoding_rs 转换
- **默认编码**：UTF-8、GBK、GB2312、Big5

### 3. Markdown 解析方案
- **选项 A**：后端 pulldown-cmark（Rust）
- **选项 B**：前端 marked.js（TypeScript）
- **建议**：前端 marked.js，减少后端复杂度

### 4. 目录提取方案
- **MD**：从 # 标题层级自动提取 TOC
- **TXT**：智能章节分割（匹配"第X章"、"Chapter X"等模式）
- **PDF**：从书签/大纲提取（如有）

## 📝 注意事项

1. **渐进式改造**：先实现 PDF/TXT/MD 核心功能，再逐步增强
2. **数据兼容**：考虑是否保留漫画数据或清空重建
3. **测试覆盖**：每个阶段都要有充分的测试，特别是不同编码的 TXT 文件
4. **性能优先**：文本渲染对性能要求高，需注意虚拟滚动和缓存
5. **编码兼容**：TXT 文件编码多样，需处理 GBK、GB2312、Big5 等中文编码
6. **PDF 差异**：PDF 文本提取可能丢失格式，需处理扫描件和无文本层的情况
